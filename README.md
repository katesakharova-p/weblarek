# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные

В приложении используются две сущности — товар и покупатель.

#### Товар (IProduct)

Интерфейс описывает структуру товара, получаемого с сервера и используемого в приложении.

``` ts
interface IProduct {
    id: string;          // Уникальный идентификатор товара
    description: string; // Описание товара
    image: string;       // URL изображения товара
    title: string;       // Название товара
    category: string;    // Категория товара
    price: number | null;// Цена товара (null для товаров, которые нельзя купить)
}
```

`IProductsResponse`- Объект, который возвращается сервером при запросе списка товаров.

``` ts
interface IProductsResponse {
  items: IProduct[];
}
```

Поля:
`items: IProduct[]` — массив товаров

#### Покупатель (IBuyer)

Интерфейс описывает данные покупателя, необходимые для оформления заказа.

``` ts
interface IBuyer {
    payment: TPayment;   // Способ оплаты ('cash' | 'card')
    email: string;       // Email покупателя
    phone: string;       // Телефон покупателя
    address: string;     // Адрес доставки
}
```

#### Заказ (IOrder)

Объект, отправляемый на сервер при оформлении заказа.

``` ts
interface IOrder {
  payment: TPayment;    // Способ оплаты (card или cash)
  email: string;        // Email покупателя
  phone: string;        // Телефон покупателя
  address: string;      // Адрес доставки
  total: number;        // Общая стоимость заказа
  items: string[];      // Массив id товаров
}
``` 

`IOrderResult` - Ответ сервера после успешного оформления заказа.

``` ts
interface IOrderResult {
  id: string;           // Уникальный идентификатор заказа
  total: number;        // Общая стоимость заказа
}
```

### Тип способа оплаты

Тип, ограничивающий возможные значения способа оплаты.

```  ts
type TPayment = 'card' | 'cash'; // Тип оплаты
```  

### Модели данных

В приложении используются три класса для работы с данными: каталог товаров, корзина и покупатель.

Отвечает за хранение и управление списком товаров, полученных с сервера, а также за хранение выбранного товара для детального просмотра. Генерирует события `products:changed` (при обновлении списка) и `product:selected` (при выборе товара для просмотра).

Конструктор:
`constructor(events: IEvents)` - принимает брокер событий для генерации уведомлений.

Поля класса:

`_selectedProduct: IProduct | null` - товар, выбранный для просмотра деталей

Методы класса:
`setProducts(products: IProduct[]): void` -  сохраняет массив товаров, полученный в параметрах, и генерирует событие `products:changed`
`getProducts(): IProduct[]` - возвращает массив всех товаров
`getProductById(id: string): IProduct | undefined` - возвращает товар по его ID
`setSelectedProduct(product: IProduct): void` -  сохраняет товар для подробного отображения и генерирует событие `product:selected`

#### Класс CartModel

Управляет корзиной покупок: хранение выбранных товаров, управление добавлением и удалением товаров, подсчет стоимости и количества, проверка наличия товаров в корзине. Генерирует событие `cart:changed` при любом изменении содержимого корзины.

Конструктор:
`constructor(events: IEvents)` - принимает брокер событий для генерации уведомлений.

Поля класса:
`_items: IProduct[]` - массив товаров в корзине

Методы класса:
`getItems(): IProduct[]` - возвращает массив товаров в корзине
`addItem(product: IProduct): void` - добавляет товар в корзину (если его там еще нет), генерирует `cart:changed`
`removeItem(product: IProduct): void` — удаляет товар из корзины, генерирует `cart:changed`
`clear(): void` - полностью очищает корзину, генерирует `cart:changed`
`getTotal(): number` - возвращает общую стоимость всех товаров в корзине (суммирует цены, игнорируя товары с ценой null)
`getCount(): number` — возвращает количество товаров в корзине
`contains(id: string): boolean` — проверяет наличие товара в корзине по его id

#### Класс BuyerModel

Хранит данные покупателя, необходимые для оформления заказа, и выполняет валидацию введённых данных покупателя. Генерирует событие `buyer:changed` при любом изменении данных

Конструктор:
`constructor(events: IEvents)` — принимает брокер событий для генерации уведомлений.

Поля класса:
`_payment: TPayment | null` — выбранный способ оплаты
`_address: string` — адрес доставки
`_phone: string` — телефон покупателя
`_email: string` — email покупателя

Методы класса:
`setData(data: Partial<IBuyer>): void` — сохраняет переданные данные покупателя (позволяет обновлять отдельные поля без перезаписи остальных)
`getData(): Partial<IBuyer>` — возвращает текущие данные покупателя
`clear(): void` - очищает все данные покупателя
`validate(): Partial<Record<keyof IBuyer, string>>` — выполняет валидацию всех полей покупателя и возвращает объект с ошибками

Отвечает за взаимодействие с сервером.

### Слой представления (View)

Слой View отвечает за отображение данных и взаимодействие с пользователем. Все классы представления работают только с DOM и не содержат бизнес-логики.

Общие принципы:
- каждый класс отвечает за свой участок разметки;
- все DOM-элементы находятся в конструкторе и сохраняются в поля;
- события навешиваются один раз в конструкторе;
- представление не хранит данные — только отображает;
- любое действие пользователя генерирует событие.

#### Класс Component

Базовый абстрактный класс для всех компонентов интерфейса. Реализует общую логику рендеринга и работы с DOM.

Конструктор:
`constructor(protected readonly container: HTMLElement)` — принимает корневой DOM-элемент компонента.

Методы:
`render(data?: Partial<T>): HTMLElement` — обновляет компонент переданными данными (через `Object.assign`) и возвращает корневой элемент.
`setImage(element: HTMLImageElement, src: string, alt?: string): void` — утилитарный метод для установки изображения с альтернативным текстом.

Абстрактный базовый класс для всех типов карточек товаров. Наследуется от Component<T>.

`constructor(container: HTMLElement)` — принимает корневой элемент карточки.

Поля:
`protected title: HTMLElement` — элемент для названия

### Слой презентера (Presenter)
3
Презентер (Presenter) является центральным компонентом, связывающим модели и представления. Он подписывается на все события приложения и реализует бизнес-логику.

Конструктор:
Принимает все необходимые зависимости: брокер событий, модели, компоненты представления и API.

Основные обязанности:
- Подписка на события (subscribeEvents)
- Отрисовка каталога (renderCatalog)
- Открытие превью товара (openPreview)
- Отрисовка корзины (renderBasket)
- Обработка изменения форм заказа (валидация через BuyerModel)
- Отправка заказа на сервер
- Очистка состояния после успешной покупки

Особенности реализации:
- Не генерирует события самостоятельно (только реагирует на них)
- Использует cloneTemplate из utils для создания DOM-элементов карточек
-  Управляет состоянием кнопки в превью через buttonState

###  События приложения

#### События моделей

`products:changed` — изменился список товаров
`cart:changed` — изменилось содержимое корзины
`buyer:changed` — изменились данные покупателя

#### События представления

`card:select` — открывает карточки товара
`basket:open`	-	открывает корзину
`order:open`	-	открывает форму заказа
`order:change`	- обновляет BuyerModel и валидирует
`order:submit`	- открывает форму контактов
`contacts:change`	- обновляет BuyerModel и валидирует
`contacts:submit`	-	отправляет заказ на сервер
`success:close`	-	закрывает модальное окно
`preview:action` -	добавляет/удаляет товар из корзины

### Инициализация приложения (main.ts)

В файле main.ts происходит начальная настройка приложения:

- Создается экземпляр EventEmitter — центральная шина событий.
- Инициализируются модели (ProductsModel, CartModel, BuyerModel) с передачей events.
- Создаются компоненты представления:
      - Header — для отображения счетчика корзины
      - CatalogView — контейнер каталога
      - Modal — для модальных окон
      - PreviewCard, BasketView, OrderForm, ContactsForm, SuccessView
- Создается экземпляр WebLarekApi для работы с сервером.
- Создается экземпляр Presenter с передачей всех зависимостей.
- Выполняется запрос к API для получения списка товаров. Полученные данные сохраняются в ProductsModel, что через событие products:changed запускает отрисовку каталога в презентере.
